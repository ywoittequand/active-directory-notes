# Pivoting

## Overview

Pivoting refers to using a compromised system as an intermediary to access additional systems or networks.

Pivoting is common during internal penetration tests and assumed breach scenarios.

---

# Objectives

* Reach segmented networks
* Access internal services
* Expand attack paths
* Access restricted assets

---

# Common Pivoting Scenarios

## Multi-Segment Networks

Accessing internal systems behind segmentation boundaries.

---

## Administrative Servers

Using management systems to access sensitive infrastructure.

---

## Hybrid Environments

Moving between on-premises and cloud-connected systems.

---

# Assessment Workflow

1. Identify reachable systems
2. Analyze segmentation boundaries
3. Review administrative relationships
4. Evaluate exposure paths

---

# Enterprise Risks

* Segmentation bypass
* Expanded attacker access
* Tier 0 exposure

---

# Defensive Considerations

* Network segmentation
* Access control reviews
* Administrative isolation
* Monitoring lateral movement activity
